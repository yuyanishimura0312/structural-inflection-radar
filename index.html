構造変化点検出レーダーとは

本ツールは、新興技術の構造的変化点（Structural Inflection Point）を早期に検出するための情報ダッシュボードです。学術論文（OpenAlex）とテックコミュニティ（HackerNews）から収集したデータを、AIが5つの軸でスコアリングし、スコアの加速度（変化の勢い）が高いテーマを「加速群」として可視化します。

絶対スコアではなく「加速度」に注目するのがこのツールの特徴です。スコアが低くても急上昇しているテーマは、今後の重要変化を示唆する先行指標になります。

画面の見方

サマリーカード（画面上部）

4つのKPIカードが全体の概況を示します。

  • 加速中テーマ：前回スキャンから加速度がプラスのテーマ数
  • 総テーマ数：全ドメインで検出されたテーマの合計
  • 分析ソース：スコアリングに使用した論文・記事の総数
  • 最終更新：パイプラインの最終実行日

ドメイン別ヒートマップ

10個のドメインごとに、加速中テーマの件数と平均加速度を表示します。セル下部の塗りの高さが平均加速度の強さを示しており、どのドメインで変化が活発かを一目で把握できます。セルをクリックすると、そのドメインで下の一覧をフィルタします。

加速度ランキング Top 10

加速度が最も高い上位10テーマを水平バーチャートで表示します。バーの長さが加速度に比例しており、最も勢いのある技術テーマを視覚的に把握できます。

加速テーマ一覧

加速度がプラスの全テーマをリスト表示します。各行には以下の情報が含まれます。

  • テーマ名：技術テーマの名称（日本語表示）
  • ドメインタグ：所属する分野（AI、エネルギー、半導体など）
  • フェーズバッジ：加速度の強さに応じた分類
  • スコア：5軸の総合スコア（0〜100%）
  • 件数：そのテーマに関連するソース記事・論文の数
  • 加速度：前回スキャンからのスコア変化率

各テーマは以下の5つの構造変化指標で評価されます。これらはAI（Claude Haiku）が学術論文やテック記事の内容を分析して判定します。

データパイプライン

本ツールのデータは以下の4段階のパイプラインで生成されます。

  1. 収集（Collect）：OpenAlex API（学術論文）とHackerNews Algolia API（テック記事）からキーワード検索で記事を取得。過去30日間、ドメインごとにキーワード別最大10件。
  2. スコアリング（Score）：Claude Haiku 4.5が各記事を5軸で分析。テーマタグの付与と各軸のシグナル/エビデンスを生成。学術論文は信頼度1.0、テックメディアは0.7で重み付け。
  3. 集約（Aggregate）：テーマタグごとにスコアを集約。前回スナップショットとの差分（デルタ）を計算。2件未満のテーマはノイズとして除外。
  4. エクスポート（Export）：集約結果をJavaScriptファイル（themes_data.js）として出力。GitHub Pagesで静的に公開。

データベース構造

バックエンドではSQLiteデータベースを使用して時系列データの蓄積を行っています。以下が主要テーブルの構造です。

domains（ドメインマスタ）

domains (10件)
├─ id INTEGER PK
├─ slug TEXT UNIQUE (例: ai_agents, energy)
├─ name_ja TEXT (日本語名)
├─ name_en TEXT (英語名)
├─ description TEXT
└─ is_active INTEGER

domain_keywords（検索キーワード）

domain_keywords (323件)
├─ id INTEGER PK
├─ domain_id INTEGER → domains.id
├─ keyword TEXT (検索キーワード)
├─ source_type TEXT (openalex / hackernews)
└─ is_active INTEGER

sources（データソース）

sources (3件)
├─ id INTEGER PK
├─ slug TEXT (openalex / hackernews / arxiv)
├─ source_type TEXT (academic / tech_media)
├─ credibility_weight REAL (学術=1.0, テック=0.7)
└─ is_active INTEGER

raw_items（収集された記事・論文）

raw_items
├─ id INTEGER PK
├─ external_id TEXT (OpenAlex/HN ID)
├─ domain_id INTEGER → domains.id
├─ source_id INTEGER → sources.id
├─ title TEXT
├─ abstract TEXT
├─ url TEXT
├─ publication_date TEXT
├─ cited_by_count INTEGER
└─ run_id INTEGER → pipeline_runs.id

scored_items（AIスコアリング結果）

scored_items
├─ id INTEGER PK
├─ raw_item_id INTEGER → raw_items.id
├─ model TEXT (claude-haiku-4-5)
├─ credibility_weight REAL
├─ threshold_proximity_signal INTEGER (0/1)
├─ threshold_proximity_evidence TEXT
├─ external_pressure_signal/evidence
├─ tech_cycle_position_signal/evidence
├─ institutional_change_signal/evidence
├─ uncertainty_resolution_signal/evidence
└─ scored_at TEXT

themes / theme_scores / theme_axes（テーマ時系列）

themes (テーママスタ)
├─ id INTEGER PK
├─ domain_id INTEGER → domains.id
├─ tag TEXT (例: perovskite_solar_cells)
└─ name_ja TEXT

theme_scores (スキャンごとのスコア)
├─ theme_id INTEGER → themes.id
├─ scan_date TEXT (UNIQUE: theme_id + scan_date)
├─ composite_score REAL
└─ composite_delta REAL

theme_axes (軸ごとのスコア)
├─ theme_score_id INTEGER → theme_scores.id
├─ axis TEXT
├─ score REAL
└─ delta REAL

pipeline_runs（パイプライン実行履歴）

pipeline_runs
├─ id INTEGER PK
├─ domain_id INTEGER → domains.id
├─ started_at / completed_at TEXT
├─ status TEXT (running / completed / error)
├─ collect_count / score_count INTEGER
├─ model TEXT
└─ estimated_cost_usd REAL

データフロー図

domains + domain_keywords
    ↓ (キーワード定義)
collect.py → raw_items
    ↓ (AI分析)
score.py → scored_items + scored_item_tags
    ↓ (テーマ集約)
aggregate.py → themes + theme_scores + theme_axes
    ↓ (JS出力)
export_js.py → themes_data.js → GitHub Pages公開

フィルタとソート

ドメインフィルターのピル（タブ）をクリックすると、特定のドメインのテーマだけを表示できます。複数選択も可能です。「全て」を押すとフィルタを解除します。ソートは右上のドロップダウンから「加速度順」「スコア順」「ソース件数順」を選択できます。

更新頻度とデータソース

パイプラインはローカル環境で手動実行されます。実行コマンドは python3 run_pipeline.py です。ドメインを指定する場合は python3 run_pipeline.py --domain energy のように指定します。収集されたデータは themes_data.js としてエクスポートされ、GitHubへのpushで自動的に公開版が更新されます。
